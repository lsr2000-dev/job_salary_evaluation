# 工作所得评估计算器

### 主题
劳动即意味着用时间换收入

一份工作的收入是其最可视化的价值，但是工作的代价对价值的损益却经常被忽视

本计算器通过对工作带来的各项收益和成本量化分析，计算出该工作<u>每分钟的货币结余</u>

用这个指标横向测评不同工作的从事价值；

### 视图

字体颜色原则：无论是金钱或时间，所有支出/赤字标记为红色，所有收入/盈余标记为绿色

网页以对比工作价值为主；用户可以点击“+”图标来添加一个新比较的工作栏目；用户需要给工作一个名

创建工作栏目后，用户需要初始化以下所有数值；

直接显示在醒目位置的数值有：

* 整数（必填数值） - <u>税前基本工资</u>
* 整数（必填数值） - <u>税前绩效工资</u>
  * 和税前基本工资合计=<u>税前月薪</u>
* 整数（必填数值） - <u>年终奖金系数</u>（年终奖金/税前基本工资=年终奖金系数）
* 选项（必填数值） - <u>每月工作日数</u>（根据选项自然计算）
  * 国家法定节假日
  * 大小周（一般周末休息两天，但每隔一个周末只休息一天）
  * 单休（每个周末只休息一天）
* 整数（必填数值）- <u>日均在岗时长</u>（原「日均工作时长」）
  * 按实际在岗小时数填写；可再填 **在岗休息（小时/日）**，工具用「在岗 − 休息」得到 **日均实际工作时长**，用于可支配效率（每分/秒）分母

其他数值在必填版面中以tab形式打开填写

有以下版面和其下的数值：

* 社会保险和公积金
  * 选项（必填数值）- <u>是否缴纳社保</u>
    * 如果选择否：不计算社会保险对每分钟收入的影响；无法填入社保基数
  * 整数（必填数值） - <u>社保基数</u>
    * 提示用户可填当地最低标准或税前基本工资
  * 选项（必填数值） - <u>是否缴纳公积金</u>
    * 如果选择否：不计算社会保险对每分钟收入的影响；无法填入：
      * 公积金缴纳基数
      * 公积金缴纳比数
  * 整数（必填数值） - <u>公积金缴纳基数</u>
    * 提示用户可填当地最低标准、税前基本工资
  * 整数（必填数值） - <u>公积金缴纳比例
    * 范围5~12，单位（单位%）
* 住房
  * 整数（必填数值） - <u>每月租金</u>
* 工作日通勤
  * 整数（必填数值） - <u>单趟通勤成本</u>
  * 整数（必填数值）- <u>单趟通勤时间</u>
* 工作日饮食
  * 整数（必填数值） - <u>日均早餐花销</u>
  * 整数（必填数值） - <u>日均午餐花销</u>
  * 整数（必填数值） - <u>日均晚餐花销</u>
  * 整数（必填数值） - <u>日均饮品零食花销</u>

在可填写数值下，对比表按 **四大区块** 展示（横向对比列=工作；行=指标）。**表头（每列工作）** 自上而下为：**国家/地区（一级经济体）**、**州/省份等二级区划**（美国税制下为 CA/TX 州选择，否则为自由文本）、**收入货币**、**税制**。从 datalist 或常见地名输入经济体时，若当前货币/税制尚未脱离上一档「建议默认」，会自动联动为该地区常用货币与税制（避免覆盖你已手改的货币或税制）。

**填写区**与**计算区**及税费下的各税制子块均可 **点击标题行折叠/展开**；折叠状态会随存档保留。

1. **收入（自动）**：基本工资、绩效、年终与额外收入等；多行金额会随顶栏 **按月 / 按日(工作日) / 按年** 切换 **月薪 / 折合工作日薪 / 年薪** 展示口径。
2. **税费（自动）**：分为 **税款**（如综合所得个税、联邦/州所得税）与 **福利费用**（五险一金个人、FICA 雇员等）分项行；企业侧说明可在行内副文案查看。
3. **就业地必要开支（自动）**：租（全月）、食与通勤（按工作日）、额外支出等。
4. **所得（自动）**：**税后收入**（与 `税前收入 − 个人扣缴` 口径一致）；**可支配所得** 可由顶栏 **所得展示** 切换密度：跟随统计周期、年、月、每工作日、每工作日·分钟、每工作日·秒（后三者由月度可支配与工作日、实际工作时长推导）。

**时间**：区分 **在岗 / 在岗休息 / 实际工作**，并保留 **在岗+通勤** 的时间付出汇总。

**报酬效率**：默认「每分钟/小时所得、收入」以 **实际工作时长** 为分母；另给出 **含通勤** 口径的行，便于对照。

### 特性

* 高交互性：用户每填写一个必填数值，就显示一个可以求值的计算值；
* 自定义：用户可以自行填写额外的收入（税后）和支出
* 对比功能：用户应该可以直观的横向对比其添加的工作，最多同时对比五个；删除和编辑工作应该方便；对比表列宽以 **320px（指标）: 330px（单列工作）** 为基准比，随页面宽度等比例缩放；一至三列时信息区宽度与三列相同、表格在区内水平居中；

### 税制模式（中国 / 美国 W-2 / 香港 / 不使用）

每列工作可在表头选择 **税制**：

* **中国**：沿用五险一金 + 综合所得个税估算（与原说明一致）。
* **美国 W-2**：简化估算 **联邦所得税（Single + 标准扣除）**、**FICA（SS + Medicare，含高收入附加）**、**州所得税**。表头在选美国时可再选 **州税** 辖区：**加利福尼亚 CA**（有州个人所得税）与 **德克萨斯 TX**（无州个人所得税），便于同薪对比税负差异。

美国模式未建模：401(k)、HSA、分项扣除、AMT、地方税、1099 自雇税等。税年与税率参数见页面内说明（当前实现以代码常量中的税年为准）。**不构成税务建议。**

* **香港**：简化估算 **雇员强积金（MPF）** 与 **薪俸税（累进税率 vs 标准税率取低；可配置基本/已婚免税额等）**。香港列为跨地区对比用估算，**不构成税务意见**；未覆盖合并评税、配偶入息、供养、物业利息等复杂情形。

* **不使用**：本列 **不计算任何税费或个人法定扣缴**；税后收入与税前收入口径一致，仅作对比占位（例如远程合同、税前包干等场景）。**不构成税务建议。**
